Web 4 Documentation

Introduction

Welcome to Web 4, the new generation of the web. Web 4 is going to enable users to use the internet much faster, more safely, and more intuitively. This documentation will provide a general overview of the architecture and features of Web 4, including how one gets started with development on the platform.

Architecture

Web 4 depends on blockchain-based decentralized infrastructure; thus, it allows peer-to-peer transactions and the sharing of data in a secure form. The following are the layers of this platform:

  • Network Layer

    • A decentralized network of nodes that allows users and applications to communicate and exchange data.

  • Blockchain Layer

    • A decentralized, secure ledger of all the various data exchanges in transactional form on the network.

  • Application Layer

    • A layer of dApps that then provide services and functionality to the end users.

Features

Web 4 offers a range of new and upcoming features that helps new and current developers like:

  • Confuscated Identify: Developers have full control over what DSB (data scraping bots) can collect from their domain.(IP, location, name, etc.) While this is great for security, scammers cannot be identified by normal people. Which is why websites have a unique decryption key to access this information, we have designed DSBs that can bypass this which can be used by FBI, trusted developers and backdooring scams.
  • Secure Transactions: Fast and secure transactions enabled by blockchain technology. Transaction details cannot be considered 'sold data'. Your website can and will be shutdown.
  • Enviorment: Web 4 ensures that you have a secure and stable environment to develop on. Web 4 uses a python/java virtual machine to run your code. This means that you can develop with computer-based code and have it run within your website, this also makes it so that you cannot infect a persons computer just by entering a website.
    • These features are not final, so some may be added later on.

Getting Started

To get started with developing on Web 4, follow these steps:

1. Install the Web 4 SDK

The Web 4 SDK provides a set of tools and libraries for building dApps on the platform.

2. Choose a Programming Language

Web 4 supports a wide range of programming languages, including Python, Java, and JavaScript

3. Build your dApp

Once you have chosen a programming language, you can start building your dApp.

4. Deploy Your dApp

Deploy your dApp on your desired website hoster.
